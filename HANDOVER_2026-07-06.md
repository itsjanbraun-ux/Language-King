## Empfohlene naechste kleine Schritte

### Schritt 1: Lernfortschritt in Supabase vorbereiten

Ziel: `learnStats` geraeteuebergreifend machen.

Vorschlag Tabelle:

```sql
create table if not exists profile_word_stats (
  id uuid primary key default gen_random_uuid(),
  profile_id uuid references profiles(id) on delete cascade,
  language text not null,
  foreign_word text not null,
  correct_count int default 0,
  wrong_count int default 0,
  last_seen timestamptz,
  unique(profile_id, language, foreign_word)
);
```

Implementationsidee:

- `saveLearnStatToDB(profile, language, word, stat)` ergaenzen.
- Beim `markLearn()` zusaetzlich Supabase upsert.
- Fallback lokal beibehalten.

### Schritt 2: Bild-Wort vorbereiten

Ziel: Datenmodell fuer Bilder erweitern, ohne sofort 200 Bilder zu brauchen.

SQL:

```sql
alter table vocab_items add column if not exists image text;
alter table vocab_items add column if not exists example text;
alter table vocab_items add column if not exists example_translation text;
alter table vocab_items add column if not exists difficulty int;
alter table vocab_items add column if not exists word_type text;
```

Dann:

- `loadVocabItemsFromDB()` um optionale Felder erweitern.
- Lokale Vokabelobjekte duerfen optional `image`, `example`, `difficulty` haben.
- Bild-Wort-Modus nur Karten mit `image` nutzen lassen.
- Wenn keine Bilder vorhanden: klare Meldung, kein leerer Screen.
