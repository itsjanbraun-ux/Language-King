### 3) Vocab from DB
- Added cache: `dbVoc`

  1. use `dbVoc` when available
  2. fallback to `localStorage`

### 4) Profiles DB sync (first pass)
- Added:
  - `SB_USER_ID`
  - `initSbUser(cb)` (reads `sb.auth.getUser()`)

  - `loadProfilesFromDB(cb)`

  - `createProf()` now calls `saveProfToDB(np)`

### 5) Stability guard
- Wrapped `endRound()` in `try/catch` with fallback to home screen and toast if a runtime error occurs.

## Update (2026-05-21)

### Done today

1. Parent-account model implemented
- Goal confirmed: only parent/main user can create/delete/manage child profiles.
- Added parent auth UI on login screen:
  - email input (`parEmail`)
  - password input (`parPass`)
  - buttons: `parentLogin()`, `parentSignup()`, `parentLogout()`
  - auth status label (`parAuthMsg`)

2. Parent auth logic added
- New globals:
  - `PARENT_AUTH`
  - `SB_USER_ID` (used for DB ownership)
- Added:
  - `syncParentAuthUI()`
  - `needParentAuth()`
  - `initAuthWatcher()` (`sb.auth.onAuthStateChange`)
- Hardened access:
  - `showForm()` blocked without parent login
  - `createProf()` blocked without parent login
  - `showAdminPin()` blocked without parent login
  - `resetProfPin()` blocked without parent login
  - `deleteProf()` blocked without parent login
- Profile "Neu" tile is rendered only when `PARENT_AUTH === true`.

3. Auth/session stability
- `initSbUser()` now sets `PARENT_AUTH` + UI state from current session.
- `initAuthWatcher()` updates UI and profile loading automatically on auth state changes.
- Added explicit click/debug logs and error handling in parent auth actions.

4. Attempts persistence (DB-first history)
- Added:
  - `saveAttemptToDB(entry)` -> writes round result to `attempts`
  - `fetchAttemptsFromDB(cb)` -> reads attempts for current profile
  - `renderHistoryV2(h)` -> active history renderer (overlay-based)
- `endRound()` now writes both:
  - local `cp.history` (fallback/cache)
  - DB row in `attempts`
- `goHistory()` now uses DB-first strategy with local fallback.
- Added `attemptsDiag()` for quick console diagnostics.

### Important behavior now
- Child profile creation/deletion requires parent login.
- Child quiz usage still works via profile PIN as before.
- History should persist across reloads via `attempts` table (to be re-verified in full flow).
